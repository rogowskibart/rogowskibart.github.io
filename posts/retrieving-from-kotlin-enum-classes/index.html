Retrieving values from Kotlin enum classes

April 19, 2021 · 2 min

Not long ago, when I was working on an Android project, I’ve noticed a problem with a Kotlin function that was using an enum. I needed to get a value from an enum class but because the data was received from an external API I needed to map it based on a value. So the enum class looked something like this:

enum class WeirdEnum(val value: Int) {
    Zero,
    One,
    Two,
    Seven,
    Eight,
    Nine,
    Ten,
    Eleven,

    companion object {
        private val values = WeirdEnum.values()
        fun getByValue(value: Int) = values.first {it.ordinal == value}
    }
}

But then when I called WeirdEnum.getByValue(7) I actually got a value of WeirdEnum.Eleven instead of WeirdEnum.Seven as was expected. Take a look:

fun main() {
    val numberOne = WeirdEnum.One
    val numberSevenByValue = WeirdEnum.getByValue(7)

    println(numberOne)			// One
    println(numberSevenByValue) // Eleven
}

Do you see the it.ordinal call inside the companion object of the WeirdEnum? That was the source of the problem. You see, everything is fine when all the values in the enum used numbers as values with no breaks between them. But here, as you can see, values between 3 and 6 are missing. How can you proceed then?

It turns out it’s an easy fix once you know where to look. All it takes is to update the companion object. I changed the it.ordinal call to a it.value but for the purposes of learning I will leave both versions just to see the differences.
